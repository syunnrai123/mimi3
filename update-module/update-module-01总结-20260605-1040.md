# update-module-01 总结：优化 WebSocket 入口 /ws 鉴权

## 1. 需求与授权

- 目标接口：`GET /ws` WebSocket 节点接入入口，位于 `mimo2api/web_service.py`
- 修改目标：为公网暴露的 WebSocket 节点入口增加可配置鉴权
- 用户授权：用户已确认“同意方案A，开始实施”
- 兼容策略：新增 `MIMO_WS_TUNNEL_KEY`，不配置时保持原兼容行为；配置后强制鉴权

## 2. 影响范围

- 接入层：`mimo2api/web_service.py`
- 鉴权工具：`mimo2api/auth.py`
- 桥接节点注入脚本：`mimo2api/bridge.py`
- 桥接脚本生成逻辑：`mimo2api/manager.py`
- 配置与文档：`env.example`、`README.md`
- 测试：`tests/test_ws_auth.py`

## 3. 实施内容

- 新增 `MIMO_WS_TUNNEL_KEY` 作为 `/ws` 专用节点接入密钥。
- `/ws` 在 `accept()` 前校验 token，校验失败时关闭连接且不加入 `state.active_clients`。
- 支持三种 token 来源：
  - `Authorization: Bearer <key>`
  - `x-ws-token: <key>`
  - `?token=<key>`
- 自动注入的桥接脚本在配置密钥时携带鉴权头。
- 兼容 `websockets` 新旧版本的 `additional_headers` / `extra_headers` 参数。
- `env.example` 和 `README.md` 已补充安全配置说明。

## 5. 残余风险

- 未配置 `MIMO_WS_TUNNEL_KEY` 时，`/ws` 仍保持兼容开放模式。
- 手工编写的外部桥接客户端若未携带 token，在开启 `MIMO_WS_TUNNEL_KEY` 后会被拒绝连接。
- `?token=` 查询参数仅作为兼容方式保留，生产环境建议优先使用请求头，避免代理日志记录 URL 时暴露 token。

## 6. 回滚方式

- 低风险回滚：删除或清空 `MIMO_WS_TUNNEL_KEY`，`/ws` 恢复兼容开放模式。
- 代码回滚：还原本次涉及文件：
  - `mimo2api/auth.py`
  - `mimo2api/web_service.py`
  - `mimo2api/bridge.py`
  - `mimo2api/manager.py`
  - `env.example`
  - `README.md`
  - `tests/test_ws_auth.py`

## 7. 复核后优化

- 将 `manager.py` 里的 bridge 注入从整行文本替换改为稳定占位符替换：
  - `"__WS_URL__"`
  - `"__WS_TOKEN__"`
- 使用 `json.dumps()` 注入 URL 和 token，避免特殊字符导致生成的远端 bridge 代码非法。
- 测试增加特殊字符覆盖，验证 URL/token 中包含引号和反斜杠时仍可生成合法代码。
- README 增加生产建议：优先通过请求头传递 WebSocket token，避免把 token 放入 URL 查询参数。
